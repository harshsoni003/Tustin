Full Example Walkthrough
Step 1 — Client uploads this Excel
Imagine Tustin Group sends this leads.xlsx:

Company Name	Contact Name	Phone	Email
Acme Corp	John Smith	555-0101	john@acme.com
BlueSky Inc	Sara Lee	555-0202	sara@bluesky.io
NovaTech	Mike Chen		mike@novatech.com
That's it — basic, incomplete data. Sales can't do much with this on a call.

Step 2 — Our API receives it

POST /upload-leads
Content-Type: multipart/form-data
Body: leads.xlsx (file)
FastAPI endpoint receives the file, openpyxl parses each row into a Python dict:


rows = [
    {"Company Name": "Acme Corp", "Contact Name": "John Smith", "Phone": "555-0101", "Email": "john@acme.com"},
    {"Company Name": "BlueSky Inc", "Contact Name": "Sara Lee", "Phone": "555-0202", "Email": "sara@bluesky.io"},
    {"Company Name": "NovaTech", "Contact Name": "Mike Chen", "Phone": "", "Email": "mike@novatech.com"},
]
Step 3 — SerpAPI enrichment (the smart part)
For each row, we query SerpAPI with a Google search:


# Row 1 example
query = "Acme Corp John Smith"
SerpAPI returns Google search results as JSON:


{
  "organic_results": [
    {
      "title": "John Smith - CEO at Acme Corp | LinkedIn",
      "link": "https://linkedin.com/in/johnsmith",
      "snippet": "CEO at Acme Corp. Enterprise software company based in Austin, TX..."
    },
    {
      "title": "Acme Corp - Enterprise Solutions",
      "link": "https://acmecorp.com",
      "snippet": "Acme Corp provides cloud-based enterprise solutions for mid-market..."
    }
  ]
}
Our logic extracts:


smart_data = {
    "Website": "https://acmecorp.com",           # from result with company domain
    "LinkedIn": "https://linkedin.com/in/johnsmith",  # from linkedin result
    "Description": "Cloud-based enterprise solutions for mid-market",  # from snippet
    "Location": "Austin, TX",                     # parsed from snippet
    "Title": "CEO"                                # parsed from LinkedIn snippet
}
Repeat for all 3 rows.

Step 4 — Enriched Excel is built
Now the data looks like this:

Company Name	Contact Name	Phone	Email	Website	LinkedIn	Title	Description	Location
Acme Corp	John Smith	555-0101	john@acme.com	acmecorp.com	linkedin.com/in/johnsmith	CEO	Cloud enterprise solutions	Austin, TX
BlueSky Inc	Sara Lee	555-0202	sara@bluesky.io	bluesky.io	linkedin.com/in/saralee	CTO	AI-powered analytics	Denver, CO
NovaTech	Mike Chen		mike@novatech.com	novatech.com		VP Sales	IoT hardware startup	San Jose, CA
Saved as leads_enriched_20260408.xlsx

Step 5 — Upload to SharePoint

# 1. Authenticate with Microsoft Graph
token = msal_app.acquire_token_for_client(scopes=["https://graph.microsoft.com/.default"])

# 2. Upload file to SharePoint document library
PUT https://graph.microsoft.com/v1.0/sites/{site-id}/drive/items/root:/Sales/Leads/leads_enriched_20260408.xlsx:/content
Authorization: Bearer {token}
Body: <file bytes>
File now lives at:


SharePoint → Tustin Group Site → Sales → Leads → leads_enriched_20260408.xlsx
Step 6 — Sales uses Copilot in Teams
Sales rep is on a Teams call with John Smith from Acme Corp. They ask Copilot:

"What do we know about Acme Corp?"

Copilot searches SharePoint (automatically indexed), finds the enriched Excel, and responds:

Acme Corp — Cloud-based enterprise solutions for mid-market, based in Austin, TX.
Contact: John Smith (CEO) — john@acme.com
Website: acmecorp.com | LinkedIn: linkedin.com/in/johnsmith

The sales rep now has context mid-call without opening any files.
